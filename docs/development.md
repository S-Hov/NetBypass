# Разработка и сборка

## Стек

- C# и .NET 10;
- WPF;
- простая MVVM-структура;
- xUnit.

## Структура решения

```text
NetBypass.sln
├── src/NetBypass.App       WPF-интерфейс и сервисные модули
├── src/NetBypass.Core      hosts, профили, DoH и сетевая диагностика
└── tests/NetBypass.Tests   модульные тесты
```

## Команды

```powershell
dotnet restore
dotnet build
dotnet test
dotnet run --project .\src\NetBypass.App
```

Для запуска интерфейса требуется Windows. Тесты Core используют временные
файлы и не изменяют системный `hosts`.

## Основные правила

- изменять только блок `NETBYPASS-BEGIN / NETBYPASS-END`;
- сохранять строки пользователя вне блока;
- создавать резервную копию до первой записи;
- проверять записанный результат;
- не доверять сохранённому флагу состояния;
- не применять изменение чекбоксов до явного сохранения;
- отклонять локальные и некорректные назначения модулей.
- не применять статический IP, не прошедший TCP/TLS-проверку.

Диагностика зависит от интерфейсов `IDohResolver` и `IEndpointProbe`, поэтому
модульные тесты не должны обращаться к реальной сети.

## Проверка изменений

Перед коммитом:

```powershell
dotnet build
dotnet test
git diff --check
```

При изменении логики `hosts` добавляйте тесты как минимум для:

- пустого состояния;
- активного блока;
- отличающегося выбора;
- повреждённых маркеров;
- сохранения пользовательских строк;
- восстановления после повреждения.
